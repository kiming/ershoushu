===============================================
==              订单处理模块文档             ==
===============================================
首先解释一下订单处理的流程
当用户A看见一本书，想要借阅，其拥有者为用户B
首先用户A将书生成订单，订单状态为1
然后用户B查看订单时发现了该订单，如果他同意
订单状态改为2，即图书已经借阅出，自动生成该订单
的应还时间为用户B确认的时间+30天。只要订单的状态
不是2，那么这本书还是可以被其他用户下单的，但是
只要有一个订单的状态被改为2，那么其他的订单状态
都将被自动改为-2.用户B也可以手动拒绝订单，拒绝后
的订单状态为-2。当然只要状态还是为1，用户A也可以
人为取消订单，取消后的状态为-1
当用户A归还图书后，用户B应该处理还书事项
用户B确认的日期为最终的归还日期，然后进入状态3
待双方都评论后进入状态4，最终结束。


-----------------------------------------------
I.生成订单
-----------------------------------------------
URL:POST  /order/create
借阅者为当前session中存储的用户
需要参数
bid		要借阅的图书的id

正确返回结果
{result: 1, data: {transaction: tran}}
其中tran是一个对象
错误的返回结果和其他错误一样的格式
